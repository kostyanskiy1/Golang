Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main
import (
	"fmt"
	"os"
)
func Foo() error {
	var err *os.PathError = nil
	return err
}
func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Программа выведет: 
<nil>
false
В функции Foo() мы из функции возвращаем ссылку на объект кастомной ошибки. 
Ссылка, возвращенная с функции, в свою очередь, равна nil, так как ошибка в функции не произошла. 
Затем происходит преобразование нашего объекта кастомной ошибки к интерфейсу error.
В поле data хранится ссылка на реальную область памяти, где лежат данные изначального
объекта (до преобразования в тип интерфейс). Собственно этот тип мы и выводим - получается <nil>.
Но когда мы сравнимаем со значением nil, то у нас идет уже сравнение интерфейса, и это
сравнение не проходит успешно, так как поле itab хранит метаданные о типе и не равняется nil.

Сравнение с nil будет успешно тогда, когда оба поля интерфейса (itab, data) равняются nil.
```
